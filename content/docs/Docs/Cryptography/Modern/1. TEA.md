# TEA 加密

这一篇来讲茶加密，这个是将数据用块来区分，然后进行加密计算的，我们来直接看代码

```c
#define DELTA 0x9E3779B9

void tea_encrypt(uint32_t v[2], const uint32_t k[4]) {
    uint32_t v0 = v[0], v1 = v[1];
    uint32_t sum = 0;
    for (int i = 0; i < 32; i++) {
        sum += DELTA;
        v0 += ((v1 << 4) + k[0]) ^ (v1 + sum) ^ ((v1 >> 5) + k[1]);
        v1 += ((v0 << 4) + k[2]) ^ (v0 + sum) ^ ((v0 >> 5) + k[3]);
    }
    v[0] = v0; v[1] = v1;
}

void tea_decrypt(uint32_t v[2], const uint32_t k[4]) {
    uint32_t v0 = v[0], v1 = v[1];
    uint32_t sum = DELTA * 32;
    for (int i = 0; i < 32; i++) {
        v1 -= ((v0 << 4) + k[2]) ^ (v0 + sum) ^ ((v0 >> 5) + k[3]);
        v0 -= ((v1 << 4) + k[0]) ^ (v1 + sum) ^ ((v1 >> 5) + k[1]);
        sum -= DELTA;
    }
    v[0] = v0; v[1] = v1;
}
```

## 参数与数据布局

| 项目   | 大小                  | 参数名 |
| ------ | --------------------- | ------ |
| 明文块 | 64 bit（2 × uint32）  | v      |
| 密钥   | 128 bit（4 × uint32） | k      |
| 轮数   | **32 轮**（固定）     |        |

## 单轮拆解

v0 我们叫作前块

```C
sum += DELTA; // 计数器
v0 += ((v1 << 4) + k[0]) ^ (v1 + sum) ^ ((v1 >> 5) + k[1]);
v1 += ((v0 << 4) + k[2]) ^ (v0 + sum) ^ ((v0 >> 5) + k[3]);
```

我写了一个小表格来看作用

| 分量             | 作用   |
| ---------------- | ------ |
| `(v1 << 4) + k0` | 扩散   |
| `(v1 + sum)`     | 轮依赖 |
| `(v1 >> 5) + k1` | 非线性 |

## 实现层面的关键特征

### 1. 常量特征

TEA 在实现层面最显著的特征是常量 `0x9E3779B9`，该值来源于黄金分割比例，
在代码中以以下形式出现：

- 单轮累加：`sum += DELTA`
- 解密初值：`sum = DELTA * 32`
  > 在实际二进制中，该常量通常不会被隐藏，是定位 TEA 的首要锚点

### 2. 指令级运算模式

无论在 x86 还是 ARM 架构下，TEA 的轮函数都会稳定表现出以下模式：

- 左移 + 右移（LSL / LSR 或 SHL / SHR）
- ADD 与 XOR 的密集交替
- 两个 32-bit 寄存器交替更新

### 3. Trace 视角下的轮结构

第一个特征就是 0x9e3779b9
第二个就是看某个寄存器是否会有间隔递增(可以找一个 timeless debugger 面板来看)
第三就是大量的前后两块数据相加、异或、左右位移

## 小结

- TEA 是一个结构极简但高度规律的分组加密
- 算法层面易懂，**实现层面特征极强**
- 在逆向与 trace 场景中非常容易被识别
  > 本文仅讨论 TEA 的核心算法实现，不涉及具体工作模式
