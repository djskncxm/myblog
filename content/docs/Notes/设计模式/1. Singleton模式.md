Singleton 就是单例模式，虽然简单但是也有很深奥的哲学

> 保证一个类在整个程序生命周期中只有一个实例，并提供全局访问点。

生命周期不只是程序周期，是你自己定义的一个周期比如：一个爬虫实例自己的整个任务规划在爬虫自己看来自己就是一个完整的程序

核心目标只有两个：

1. **唯一实例（Only One Instance）**
2. **全局访问（Global Access Point）**

典型用途：

- 配置管理器
- 日志系统
- 线程池
- 数据库连接池
- 全局状态管理
  比如

```C++
Config::GetInstance().Get("db_host");
Logger::GetInstance().Info("hello");
```

我们来用 C++写一个例子

```C++
// 无锁版本
class Singleton {
private:
        static Singleton* instance;

        Singleton() {} // 禁止外部创建

public:
        static Singleton* GetInstance() {
                if (instance == nullptr) {
                        instance = new Singleton();
                }
                return instance;
        }
};

Singleton* Singleton::instance = nullptr;

// 有锁版本
#include <mutex>

class Singleton {
private:
        static Singleton* instance;
        static std::mutex mtx;

        Singleton() {}

public:
        static Singleton* GetInstance() {

                if (instance == nullptr) {

                        std::lock_guard<std::mutex> lock(mtx);

                        if (instance == nullptr) {
                                instance = new Singleton();
                        }
                }

                return instance;
        }
};

Singleton* Singleton::instance = nullptr;
std::mutex Singleton::mtx;
```

整体逻辑也就是会下面这样

```
第一次调用 GetInstance()
↓
instance == null
↓
创建对象
↓
后续所有调用返回同一个对象
```

单例模式的本质就是全局静态变量+控制初始化

```
Program
   │
   ├── global Singleton*
   │
   └── GetInstance()
            │
            ├── 未初始化 → 创建
            └── 已初始化 → 返回
```

单例虽然简单明了，但是也有不好的地方，

1. 全局变量污染
2. 测试的时候很不好测试
3. 内部隐式依赖，你也不知道这个里面有多少逻辑

那么在什么地方使用呢？

1.  确实只有一个，日志系统，配置文件这种确实只有一个的内容可以在适当使用
2.  全局资源，全部都从某一个地方进行获取资源(内存，线程，连接)情况
3.  创建一次的代价极其昂贵(数据库每次创建和断开连接)

使用的时候理解其实也很简单

1. 谁创建
2. 谁销毁
3. 是否只需要一个
