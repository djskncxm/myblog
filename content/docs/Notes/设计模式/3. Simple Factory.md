简单工厂是拓展工厂方法的一种思路，先来看简单工厂本身，最后再进行比较

首先我们有 N 个类，并且这些类都有一定的相似性，比如动物

```C++
class Dog {};
class Cat {};
class Bird {};
```

这样写我们是需要写挨个新建，有使用到就都需要改，这时候我们就可以使用一种思路，将这些类集中管理，这就是简单工厂

这个是基类

```C++
class Animal {
public:
    virtual void speak() = 0;
    virtual ~Animal() = default;
};
```

这个是具体类

```C++
class Dog : public Animal {
public:
    void speak() override {
        std::cout << "Dog bark\n";
    }
};

class Cat : public Animal {
public:
    void speak() override {
        std::cout << "Cat meow\n";
    }
};
```

简单工厂类

```C++
class AnimalFactory {
public:
    static Animal* create(const std::string& type) {
        if (type == "dog")
            return new Dog();

        if (type == "cat")
            return new Cat();

        return nullptr;
    }
};
```

使用的时候

```C++
int main() {
    Animal* a = AnimalFactory::create("dog");
    a->speak();
}
```

由例子可以知道简单工厂的核心结构

| 角色            | 职责                 |
| --------------- | -------------------- |
| Product         | 产品接口             |
| ConcreteProduct | 具体产品             |
| Factory         | 工厂类，负责创建对象 |

> **把对象创建逻辑集中到一个类里**

当然，这时候我们会遇见工厂越来越大的，后面整个文件都是工厂的判断语句，这时候就需要知道前人是怎么解决的

> **注册表思路**

```C++
using Creator = std::function<Animal*()>;

class AnimalFactory {
public:
    static Animal* create(const std::string& type) {
        return registry()[type]();
    }

    static void registerAnimal(const std::string& type, Creator c) {
        registry()[type] = c;
    }

private:
    static std::unordered_map<std::string, Creator>& registry() {
        static std::unordered_map<std::string, Creator> r;
        return r;
    }
};
```

注册出来类，然后工厂内部维护一个 map 或者 table，然后用的时候进行创建
