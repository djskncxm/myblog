## 1️⃣ 概念

Builder 模式是一种**创建型设计模式**，主要用于构建复杂对象。  
它的核心思想是**将对象的构建过程与表示分离**，使同样的构建过程可以创建不同的表示。

### 适用场景

- 对象构建复杂，参数多且可选项多
- 需要隔离复杂对象内部实现
- 想用统一流程生成不同配置的对象

```
     +------------+                 |
     | buildPartA()                  |
     | buildPartB()                  |
     | getResult() -----------------+
     +------------+
           ^
           |

+-----------------+
 | ConcreteBuilder |
 +-----------------+
 | buildPartA() |
 | buildPartB() |
 | getResult() |
 +-----------------+
 |
 v
 +--------+
 | Product|
 +--------+
 | partA |
 | partB |
 +--------+
```

- **Product**：最终构建的对象
- **Builder**：定义构建接口
- **ConcreteBuilder**：具体实现
- **Director**（可选）：按顺序指挥 Builder 构建

## 3️⃣ Java 示例

```java
// 产品
class Burger {
    private String bun;
    private String meat;
    private boolean cheese;

    public void setBun(String bun) { this.bun = bun; }
    public void setMeat(String meat) { this.meat = meat; }
    public void setCheese(boolean cheese) { this.cheese = cheese; }

    @Override
    public String toString() {
        return "Burger [bun=" + bun + ", meat=" + meat + ", cheese=" + cheese + "]";
    }
}

// 抽象建造者
interface BurgerBuilder {
    void buildBun();
    void buildMeat();
    void buildCheese();
    Burger getResult();
}

// 具体建造者
class BeefBurgerBuilder implements BurgerBuilder {
    private Burger burger = new Burger();

    public void buildBun() { burger.setBun("Sesame Bun"); }
    public void buildMeat() { burger.setMeat("Beef"); }
    public void buildCheese() { burger.setCheese(true); }
    public Burger getResult() { return burger; }
}

// 指挥者（可选）
class BurgerDirector {
    private BurgerBuilder builder;
    public BurgerDirector(BurgerBuilder builder) { this.builder = builder; }
    public Burger construct() {
        builder.buildBun();
        builder.buildMeat();
        builder.buildCheese();
        return builder.getResult();
    }
}

// 客户端
public class Main {
    public static void main(String[] args) {
        BurgerBuilder builder = new BeefBurgerBuilder();
        BurgerDirector director = new BurgerDirector(builder);
        Burger burger = director.construct();
        System.out.println(burger);
    }
}
```

## 4️⃣ 浏览器配置场景示例（JS/TS）

在浏览器自动化中，我们通常需要配置：

- 浏览器路径
- 启动参数（headless、窗口大小）
- 代理
- CDP 端口
- 用户数据目录

Builder 模式可以让这些配置清晰可控：

```js
// 产品类
class BrowserConfig {
    path?: string;
    headless?: boolean;
    proxy?: string;
    cdpPort?: number;
    userDataDir?: string;
}

// Builder 类
class BrowserBuilder {
    private config = new BrowserConfig();

    setPath(path: string) {
        this.config.path = path;
        return this;
    }

    setHeadless(headless: boolean) {
        this.config.headless = headless;
        return this;
    }

    setProxy(proxy: string) {
        this.config.proxy = proxy;
        return this;
    }

    setCdpPort(port: number) {
        this.config.cdpPort = port;
        return this;
    }

    setUserDataDir(dir: string) {
        this.config.userDataDir = dir;
        return this;
    }

    build() {
        if (!this.config.path) throw new Error("Browser path must be set!");
        return this.config;
    }
}

// 客户端使用
const config = new BrowserBuilder()
    .setPath("/usr/bin/chromium")
    .setHeadless(true)
    .setProxy("http://127.0.0.1:1080")
    .setCdpPort(9222)
    .setUserDataDir("/tmp/duckbrowser")
    .build();

console.log(config);
```

**特点：**

- 链式调用，易读
- 可选项灵活
- 集中构建逻辑，可加校验或默认值

## 5️⃣ 总结

**优点：**

1. 构建过程和表示分离
2. 支持一步步构建复杂对象
3. 可复用构建流程生成不同表现

**缺点：**

1. 增加类的数量
2. 对简单对象可能过度设计

**典型应用：**

- Java 的 `StringBuilder`
- Android 的 `AlertDialog.Builder`
- 浏览器自动化配置
- 流式 API（OkHttp、CDP 驱动等）
