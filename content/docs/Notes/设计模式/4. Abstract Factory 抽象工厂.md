# 抽象工厂模式（Abstract Factory）

## 1️⃣ 概念

抽象工厂模式是一种**创建型设计模式**，用于提供一个接口，创建一组**相关或相互依赖的对象**，而无需指定它们具体类。

- 关注点：**整个产品族的创建**，而不仅仅是单个产品。
- 客户端只依赖抽象接口，不直接依赖具体类。

---

## 2️⃣ 生活化比喻：拉面店

| 阶段                              | 比喻                   | 模式概念                     |
| --------------------------------- | ---------------------- | ---------------------------- |
| 看见拉面模型                      | 只是知道有拉面，吃不到 | 抽象产品（Abstract Product） |
| 看到风味选项（香辣、麻辣、牛油…） | 可以看到、选择风味     | 具体产品（Concrete Product） |
| 点餐，厨师制作                    | 厨师根据选择制作拉面   | 具体工厂（Concrete Factory） |
| 上菜并吃                          | 客户端吃到拉面         | 客户端使用（Client）         |

💡 核心思想：

> 抽象产品定义接口，具体产品实现接口，具体工厂生产产品，客户端使用产品。

---

## 3️⃣ UML 结构

```
AbstractFactory
    ├── createProductA()
    └── createProductB()

ConcreteFactory1
    ├── createProductA() -> ConcreteProductA1
    └── createProductB() -> ConcreteProductB1

ConcreteFactory2
    ├── createProductA() -> ConcreteProductA2
    └── createProductB() -> ConcreteProductB2

ProductA / ProductB (抽象产品)
ConcreteProductA1 / A2 / B1 / B2 (具体产品)
```

## 4️⃣ Python 示例代码

```python
# ===== 抽象产品 =====
class Button:
    def paint(self):
        pass

class Checkbox:
    def paint(self):
        pass

# ===== 具体产品 =====
class WindowsButton(Button):
    def paint(self):
        print("Rendering Windows button")

class MacButton(Button):
    def paint(self):
        print("Rendering Mac button")

class WindowsCheckbox(Checkbox):
    def paint(self):
        print("Rendering Windows checkbox")

class MacCheckbox(Checkbox):
    def paint(self):
        print("Rendering Mac checkbox")

# ===== 抽象工厂 =====
class GUIFactory:
    def create_button(self) -> Button:
        pass
    def create_checkbox(self) -> Checkbox:
        pass

# ===== 具体工厂 =====
class WindowsFactory(GUIFactory):
    def create_button(self):
        return WindowsButton()
    def create_checkbox(self):
        return WindowsCheckbox()

class MacFactory(GUIFactory):
    def create_button(self):
        return MacButton()
    def create_checkbox(self):
        return MacCheckbox()

# ===== 客户端 =====
def render(factory: GUIFactory):
    button = factory.create_button()
    checkbox = factory.create_checkbox()
    button.paint()
    checkbox.paint()

# ===== 使用 =====
render(WindowsFactory())
render(MacFactory())
```

输出：

```
Rendering Windows button
Rendering Windows checkbox
Rendering Mac button
Rendering Mac checkbox
```

---

## 5️⃣ 总结与对比

- **简单工厂**：创建单个对象，客户端无需关心具体类。
- **工厂方法**：每个工厂负责创建一个产品，关注单一产品的多实现。
- **抽象工厂**：创建一整套相关产品（产品族），客户端只依赖抽象接口，保证风格一致。

---

💡 延伸思考：

- 抽象工厂适合 **产品族多、风格需要统一** 的场景，例如：UI 组件库、跨平台 SDK、主题套餐系统。
- 如果只是创建单个对象，工厂方法或简单工厂就够了。
