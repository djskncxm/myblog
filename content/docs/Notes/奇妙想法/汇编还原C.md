# 有向图类解析思路
### **第一步：线性表索引（Function List）**
- 直接线性扫描 trace，记录每条 `stp/add/ret` 等指令的函数入口、出口
- 生成 **函数索引表**（线性表即可）
  - 函数名 / 起始偏移 / 结束偏移 / 执行次数
- 简单高效，先把全局函数目录建立起来
- 核心作用：给后续调用树构建提供基础节点

---


### **第二步：树结构构建（Call Graph / Branching）**
- 遍历 trace 中的 `bl/blr` 指令，建立 **树形结构**
  - 每个函数节点连接被调用的函数
  - 可以标记重复调用次数或循环调用
- 树结构方便：
  - 快速查找调用路径
  - 统计循环或热点函数
  - 后续可直接做循环压缩和热点分析

---


### **第三步：随意分析（Loop / Semantic）**
- 对函数内部或者循环体可以随意处理
  - 只记录调用次数 / 循环次数即可
  - 不必展开每条汇编指令
- 核心是压缩和抽象，不追求完整 SSA 或 IR
- 可以结合一些 heuristics（寄存器模式、内存访问规律）做简单标注

---


### **第四步：汇总逻辑（Visualization / Analysis）**
- 最终将三个阶段的数据汇总：
  - 线性表 → 函数目录
  - 树结构 → 调用关系 + 循环识别
  - 第三步 → 循环次数/热点函数/算法关键点
- 输出：
  - JSON / 数据库结构，便于按需查询
  - 可视化调用图、循环图、关键函数分析
- 优势：
  - 可操作性强，处理大规模 trace 不会爆炸
  - 先粗后细，按需深入

---



# 🌀 回型嵌套思路（Trace → 伪 C 自动化还原）

## 1️⃣ 核心理念
- **整体视角**：将整个 trace 当作一个完整程序流，而不是孤立函数序列。
- **动态栈驱动**：利用动态执行栈模拟函数调用和返回行为，而不依赖静态 `ret` 或精确 CFG。
- **自底向上生成**：先复原最深层嵌套的子函数，再逐层嵌入父函数，保证逻辑层次清晰。
- **回型嵌套**：类似递归下降解析，通过“深入 → 回退 → 兄弟函数扫描”实现层级嵌套。

---


## 2️⃣ 核心步骤

### 2.1 函数入口识别
- **可靠入口**：
  - **`bl` 指令目标** → 静态安全、可直接当作函数入口。
  - 程序入口地址（`_start`、`main` 等）。
  - （可选）函数 prologue 特征（`stp x29, x30` 等）作为二次确认。
- **不当入口**：
  - `blr`、`br`、`b` → 不直接作为函数入口，`blr`/`br`仅用于调用边或 trace 分析。


### 2.2 区间划分与栈管理
- **栈模拟函数调用**：
  - 遍历 trace，遇到 `bl` → 入栈，标记子函数开始。
  - 遇到 `ret` 或 `bl` 地址+4 → 出栈，标记函数结束。
- **递归解析子函数**：
  - 在栈顶子函数解析完成后，返回父函数继续扫描兄弟函数。
  - 子函数结束后，父函数扫描剩余区间以发现其他兄弟函数入口。
- **区间判定原则**：
  - `bl` 的目标地址为函数入口。
  - 父函数区间可从本函数开始地址到下一个 `bl` 地址+4（或 ret）结束。


### 2.3 自底向上嵌套生成
- **生成顺序**：
  1. 先生成最深层子函数的伪 C 代码。
  2. 嵌入调用它的父函数的对应调用点。
  3. 父函数扫描剩余入口，生成同级兄弟函数。
- **优势**：
  - 内层逻辑先固定，外层调用关系自然形成嵌套。
  - 对尾调用、inline、跳转等不精确指令也能容错。

---


## 3️⃣ 调用关系与伪 C 构建
- **调用树 / 调用图**：
  - 节点 = 函数
  - 边 = 调用关系（直接 `bl` 或间接 `blr`）
- **动态调用栈驱动嵌套**：
  - 子函数先生成 → 嵌入父函数 → 父函数扫描剩余入口 → 生成兄弟函数。
- **结果**：
  - 伪 C 结构清晰，内层函数逻辑独立，外层父函数结构完整。
  - 调用图与嵌套结构一一对应。

---


## 4️⃣ 优化与扩展
- **调用边记录**：
  - `blr` / `br` → 间接调用或尾调用，仅用于构建调用图，不作为入口。
- **Trace 执行优先**：
  - 如果有 trace 数据，可在运行时解析 `blr`、`br` 的实际目标 → 自动解析动态调用边。

---


## 5️⃣ 总结原则
1. **函数入口识别只依赖可靠指令**：`bl`、程序入口、可选 prologue。
2. **调用关系完整记录**：包括 `bl`（直接）和 `blr`（间接）指令。
3. **自底向上嵌套生成**：内层函数先生成 → 嵌套父函数 → 兄弟函数扫描。
4. **动态栈 + 回型遍历**：保证函数逻辑层次清晰、完整，不依赖静态 CFG。
5. **容错性强**：尾调用、inline、跳转等不精确指令不会破坏嵌套逻辑。
