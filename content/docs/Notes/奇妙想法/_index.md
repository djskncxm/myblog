### so 不落地，无 elf 头

LSP 或者 kpm 模块在系统启动的时候就进行 so 的加载，进行无痕注入，但是需要想 LSP 和 kpm 的痕迹
不过也可以选择直接吃现成的，报个检测课，然后看手子怎么写的

### 为一个设备维护 LineageOS 分支

随便一个分支维护起来，然后写简历，找一个 ROM 开发对抗的岗位上班
看看能不能针对此设备进行隐藏注入

### 注入框架

分析 Frida、LSP 等框架看它们的 hook 是怎么实现的，尤其是 Frida 的 gatget，然后像前面说的不落地加载一样加载起来
看看怎么自己研究一个针对某个 app 的东西

使用apatch写内核模块，根据pid查找后注入so(现在思考是自己做一个renef)，根据偏移进行硬件断点，硬件无痕hook
