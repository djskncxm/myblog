对于hash先写总结吧，具体的算法内容后面再详细描写


| 哈希算法                                | 分块大小（Block size）             |
| ----------------------------------- | ---------------------------- |
| **MD5**                             | 512 bit（64 字节）               |
| **SHA-1**                           | 512 bit（64 字节）               |
| **SHA-256 / SHA-224**               | 512 bit（64 字节）               |
| **SHA-512 / SHA-384 / SHA-512/256** | 1024 bit（128 字节）             |
| **BLAKE2s**                         | 512 bit                      |
| **BLAKE2b**                         | 1024 bit                     |
| **SHA-3 (Keccak)**                  | block size 不固定（“吸收率”取决于输出长度） |
| **SM3（国密）**                         | 512 bit                      |
| **RIPEMD160**                       | 512 bit                      |

首先就是分块，我给各位出一个表格来举例子
> 传统哈希像 MD5 / SHA-1 / SHA-256 / SM3 通常用 512 bit 块  
> SHA-512 家族用 1024 bit 块  
> SHA-2 完全是另一种结构，不以固定 block size 为主    


- SHA-512 为什么变成 1024 bit？
    - 因为 SHA-512 是 64-bit CPU 友好设计：
    - 一个 block = 16×64-bit = 1024 bit
    - 内部寄存器是 64-bit
    - 一次能处理更多数据 → 更快

- SHA-3（Keccak）更不同

> SHA-3 是海绵结构（Sponge construction）。

- 它不是：
    > “把消息切成固定 512 bit 然后压缩”
- 而是：
    > 定义一个大状态（1600 bit），每次吸输入的一部分（称为 rate），然后搅拌。
    - 例如 SHA3-256：
    - 状态 1600 bit
    - rate = 1088 bit
    - 每次处理 1088 bit，不是 512。当然也不是固定的，只和输出长度相关
